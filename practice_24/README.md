# Практика 24

## Отчет по выполненным практикам 19-23

### Практика 19-20
Выполнены два backend-варианта одного учебного API:
- `backend_pg` на PostgreSQL (`pg`), структура на TypeScript + Express;
- `backend_mongo` на MongoDB (`mongoose`), структура на TypeScript + Express.

Реализована базовая модульная структура проекта:
- `src/config` для переменных окружения и подключения к БД;
- `src/app` для приложения и маршрутов;
- `src/server.ts` и `src/index.ts` для запуска сервера.

### Практика 21
Реализовано кэширование на Redis в backend-проекте:
- подключение Redis-клиента;
- сервис кэша (`src/cache/cache.service.ts`);
- подготовка логики для использования кэш-слоя в API.

Также в проекте присутствуют:
- middleware для авторизации, ролей, валидации и логирования;
- Swagger-документация (`src/docs/*`).

### Практика 22
Реализована балансировка нагрузки между несколькими backend-сервисами через два варианта прокси:
- Nginx (`docker-compose.nginx.yml`, `nginx/nginx.conf`);
- HAProxy (`docker-compose.haproxy.yml`, `haproxy/haproxy.cfg`).

Backend-сервис контейнеризирован:
- создан `backend/Dockerfile`;
- сервис отвечает информацией об экземпляре, что позволяет видеть распределение трафика.

### Практика 23
Реализовано многоконтейнерное приложение в Docker Compose:
- два идентичных backend-сервиса на Node.js (Express);
- Nginx как балансировщик нагрузки;
- отдельные файлы `docker-compose.yml`, `backend/Dockerfile`, `nginx/nginx.conf`.

Настроена базовая отказоустойчивость Nginx:
- для upstream-серверов заданы `max_fails` и `fail_timeout`;
- предусмотрена проверка сценария остановки одного backend-контейнера с продолжением обслуживания запросов.

## Итог
За практики 19-23 последовательно реализованы:
- работа с разными БД (PostgreSQL и MongoDB);
- кэширование на Redis;
- контейнеризация backend-сервисов;
- балансировка и отказоустойчивость в Docker-окружении.
