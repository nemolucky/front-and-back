# Отчёт по практическим занятиям №13–17
### Frontend & Backend разработка
**Тема:** PWA, Service Worker, HTTPS, WebSocket, Push-уведомления и напоминания

---

## Обзор

В рамках практик №13–17 разработано веб-приложение **«Заметки»**, которое:

- работает в офлайн-режиме;
- поддерживает установку как PWA;
- синхронизирует события между клиентами в реальном времени;
- отправляет push-уведомления;
- поддерживает напоминания и их отложение на 5 минут.

---

## Практическая №13 — Базовое приложение заметок

> **Цель:** Реализовать основу клиентского приложения для работы с заметками.

### Что реализовано:
- интерфейс добавления и просмотра заметок;
- сохранение заметок в `localStorage`;
- отображение списка заметок на клиенте.

---

## Практическая №14 — PWA-основа

> **Цель:** Подготовить приложение к работе как Progressive Web App.

### Что реализовано:
- подключён `manifest.json`;
- добавлены иконки приложения (`16x16 ... 512x512`);
- подключён `Service Worker`;
- добавлены PWA-мета-теги для мобильных платформ.

---

## Практическая №15 — HTTPS + App Shell

> **Цель:** Обеспечить безопасное соединение и быстрый старт интерфейса.

### Что реализовано:
- локальный запуск приложения по **HTTPS**;
- сервер использует сертификаты:
  - `localhost+2.pem`
  - `localhost+2-key.pem`
- реализована архитектура **App Shell**;
- для статики используется `Cache First`;
- для динамического контента (`/content/*`) используется `Network First`.

---

## Практическая №16 — WebSocket + Push

> **Цель:** Добавить real-time функциональность и push-уведомления.

### Что реализовано:
- сервер на `Node.js + Express`;
- интеграция `Socket.IO` для событий в реальном времени;
- рассылка события о новых заметках между вкладками/клиентами;
- push-подписка и отписка через API:
  - `POST /subscribe`
  - `POST /unsubscribe`
- отправка push через `web-push` и VAPID-ключи;
- управление уведомлениями из интерфейса (включить/отключить).

---

## Практическая №17 — Детализация Push (напоминания)

> **Цель:** Реализовать отложенные напоминания и действие «Отложить на 5 минут».

### Что реализовано:
- отдельная форма заметки с напоминанием (`text + datetime-local`);
- структура заметки в `localStorage` содержит:
  - `id`
  - `text`
  - `reminder` (timestamp)
- на сервере реализовано планирование напоминаний через `setTimeout`;
- активные таймеры хранятся в `Map`;
- добавлен endpoint:
  - `POST /snooze?reminderId=...`
- push-уведомление для напоминания содержит action-кнопку:
  - **«Отложить на 5 минут»**;
- по нажатию action сервер переносит напоминание на 5 минут.

## Технологии проекта

| Компонент | Использовано |
|-----------|--------------|
| Клиент | HTML, CSS, JavaScript |
| Офлайн-режим | Service Worker, Cache Storage API |
| PWA | Web App Manifest |
| Сервер | Node.js, Express |
| Реальное время | Socket.IO |
| Push-уведомления | Web Push API (`web-push`) |
| Авторизация push-сервера | VAPID keys |
| Локальный HTTPS | `https` (Node.js) + локальные `.pem` сертификаты |
| Хранение клиентских данных | `localStorage` |

## Запуск проекта

1. Установить зависимости:

```bash
npm install
```

2. Запустить сервер:

```bash
npm start
```

3. Открыть приложение:

```text
https://localhost:3001
```

---

## Проверка функционала

1. Открыть приложение и включить push-уведомления.
2. Добавить обычную заметку — событие должно появиться у подключённых клиентов.
3. Добавить заметку с напоминанием в будущем времени.
4. Дождаться push-уведомления с кнопкой **«Отложить на 5 минут»**.
5. Нажать кнопку и убедиться, что напоминание перенесено.
6. Проверить офлайн-режим (отключить сеть и обновить страницу).
